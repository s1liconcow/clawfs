Getting Started

Install The OSS CLI And Mount A Workspace

The public repository is now focused on the filesystem engine, the FUSE mount path, and the checkpoint and replay utilities around it.

Linux / macOS: curl -fsSL https://clawfs.dev/install.sh | bash Windows (PowerShell): iwr https://clawfs.dev/install.ps1 -UseBasicParsing | iex
Mount A Workspace Review Config

Step 1

Inspect The CLI

Confirm The Install

clawfs --help clawfs_checkpoint --help clawfs_replay --help

The public repo installs the main filesystem binary plus checkpoint and replay tools.

Prepare Local Paths

mkdir -p ~/.clawfs/store ~/.clawfs/cache /tmp/clawfs-mnt

ClawFS keeps object data, local cache, and client state in explicit paths you control.

Step 2

Mount The Filesystem

Local Quick Start

clawfs \ --mount-path /tmp/clawfs-mnt \ --store-path ~/.clawfs/store \ --local-cache-path ~/.clawfs/cache \ --state-path ~/.clawfs/state.bin \ --foreground

Run in the foreground first so you can inspect logs and confirm the mount behavior.

Backed By Object Storage

clawfs \ --mount-path /tmp/clawfs-mnt \ --object-provider aws \ --bucket my-clawfs-bucket \ --object-prefix team-a/demo \ --local-cache-path ~/.clawfs/cache \ --state-path ~/.clawfs/state.bin

Swap the backing store without changing the filesystem interface seen by tools.

Configuration

Key Runtime Knobs

Storage

--object-provider local|aws|gcs --bucket my-bucket --endpoint https://s3.amazonaws.com --object-prefix workspaces/demo

Client State

--local-cache-path ~/.clawfs/cache --state-path ~/.clawfs/state.bin --segment-cache-bytes 536870912 --pending-bytes 268435456

Durability

--disable-journal --fsync-on-close --flush-interval-ms 5000 --lookup-cache-ttl-ms 5000

Use the public repo for the filesystem itself.

Hosted business features now live outside the OSS tree, while the mount and storage core remain here.
