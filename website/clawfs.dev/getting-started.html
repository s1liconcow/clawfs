Getting Started

Launch ClawFS the Way Your Agents Actually Need It

ClawFS can be introduced at different layers depending on the runtime. The current repo ships concrete FUSE and NFS paths today, and the planned shared-library mode is the path for tighter in-process interception later.

Start With FUSE Use NFS Configure Runtime

Current Shipping Modes

  • FUSE mount via `clawfs` or `scripts/run_clawfs.sh`
  • User-mode NFS server via `clawfs-nfs-gateway` or `scripts/run_nfs_gateway.sh`

Planned Mode

  • Shared-library path for in-process filesystem interception under configured ClawFS prefixes
  • Best fit for agent launchers that want file APIs without requiring a host mount

Mode 1

FUSE: Mount ClawFS Directly on the Host

When to Use It

Use FUSE when the agent or developer environment wants a real mount point and the host can support it.

  • Local development
  • Single-node agent runners
  • Workflows that expect POSIX paths directly on the host

Quick Start

cargo build --release --bin clawfs ./scripts/run_clawfs.sh

Defaults mount to `/tmp/clawfs-mnt` and uses a local backing store under `/tmp/clawfs-store`.

Direct CLI launch
Use the binary directly if you want full control over flags:
cargo run --release --bin clawfs -- \ --mount-path /tmp/clawfs-mnt \ --store-path /tmp/clawfs-store \ --local-cache-path ~/.clawfs/cache \ --state-path ~/.clawfs/state/client_state.bin \ --object-provider local \ --foreground

Mode 2

NFS: Serve ClawFS Over User-Mode NFS

When to Use It

Use NFS when you want ClawFS running as a user-space server and the client side can mount NFS or connect from another runtime.

  • Remote workers and VMs
  • Containerized runners
  • Cases where you want a network boundary between control plane and client mount

Quick Start

cargo build --release --manifest-path clawfs-nfs-gateway/Cargo.toml ./scripts/run_nfs_gateway.sh

Default listen address is `0.0.0.0:2049`; the helper can also auto-mount the export locally when the host supports it.

Direct gateway launch
If you want to manage the process yourself:
cargo run --release --manifest-path clawfs-nfs-gateway/Cargo.toml -- \ --mount-path /tmp/clawfs-mnt \ --store-path /tmp/clawfs-store \ --local-cache-path ~/.clawfs/cache \ --state-path ~/.clawfs/state/nfs_gateway_state.bin \ --listen 0.0.0.0:2049 \ --protocol v3

Mode 3

Shared Library: Planned In-Process Path for Agents

Status: planned / not yet shipped in this repo.
The intended model is a preloadable shared library that intercepts filesystem calls for configured ClawFS paths and redirects them into ClawFS core logic. This is the direction for agents that want filesystem semantics without requiring a host-level mount.

Target UX

LD_PRELOAD=libclawfs_preload.so \ CLAWFS_PREFIXES=/claw,/agent-data \ sprite exec -- my-agent

On macOS the equivalent path would use `DYLD_INSERT_LIBRARIES` where allowed.

Why It Exists

  • No kernel mount requirement inside the agent runtime
  • Path-transparent use of normal file APIs
  • Better fit for tightly controlled agent launchers

Configuration

Configure Storage, Credentials, and Runtime Behavior

Storage Provider

Use `--object-provider local|aws|gcs`, then set the backing location with bucket, region, endpoint, and prefix controls.

--object-provider aws --bucket my-clawfs-bucket --region us-west-2 --endpoint https://s3.amazonaws.com --object-prefix agents/team-a

Client State + Cache

Each client keeps local cache and identity state. Override these when you want isolated runners or explicit paths.

--local-cache-path ~/.clawfs/cache --state-path ~/.clawfs/state/client_state.bin --segment-cache-bytes 536870912 --pending-bytes 268435456

Durability + Performance

ClawFS exposes the main durability and cache knobs directly on the CLI instead of hiding them in a daemon config file.

--disable-journal --fsync-on-close --flush-interval-ms 5000 --lookup-cache-ttl-ms 5000 --dir-cache-ttl-ms 5000

Hosted + BYOB Modes

Environment-Driven ClawFS Runtime Spec

Hosted Free Tier Shape

The runtime can be configured by env vars for hosted-free or BYOB issuance flows.

CLAWFS_STORAGE_MODE=hosted_free CLAWFS_OBJECT_PROVIDER=aws CLAWFS_BUCKET=clawfs-free CLAWFS_OBJECT_PREFIX=free/account-a/volume-1 CLAWFS_MAX_PENDING_BYTES=268435456 CLAWFS_MAX_SEGMENT_CACHE_BYTES=536870912

Scoped Credentials

For AWS-compatible object stores, ClawFS can consume scoped runtime credentials directly from env.

CLAWFS_AWS_ACCESS_KEY_ID=... CLAWFS_AWS_SECRET_ACCESS_KEY=... CLAWFS_AWS_SESSION_TOKEN=...
